## MOD-003: Style Calibration Revision

**Date:** October 19, 2025

**Issue Identified:**

User provided paper excerpts (Introduction, Section IX) and requested: "consider whether the style needs tuning based on this paper excerpts"

**Problem Identified:**

Initial draft had significant stylistic inconsistencies with main paper:

**Excessive formatting:**
- Heavy bolding throughout (paper uses bold very sparingly)
- Over-enumeration in flowing prose ("First, Second, Third" structures)
- Lists and bullets where main paper uses integrated prose

**Tone mismatch:**
- Too instructional ("you must," "the reviewer should")
- Not analytical/descriptive enough (main paper is careful, precise exposition)

**BUT:** Appendix IS reference material (not argumentative prose) - some structure appropriate for usability.

**User Confirmation:**

Agreed with assessment. Requested:
- Reduced bolding
- Smoother prose in A.2 (graph description narrative)
- Less enumeration in flowing text
- Maintained structure in reference sections (type definitions, tables) where appropriate

**Decision/Resolution:**

**Systematic style revision across all sections:**

**Reduced formatting:**
- Removed most bolding (kept only for critical phase markers: "Phase 1: Pre-Writing")
- Removed bold from type names in running text
- Kept bold for section headers and table headers only

**Eliminated enumeration in narrative sections:**
- Changed "First, Second, Third" structures to flowing prose
- Converted numbered lists in A.2 to integrated paragraphs with transitions
- Maintained numbered/bulleted lists only in A.3 (type definitions) and tables

**Changed instructional to descriptive language:**
- "You must" → "serves as"
- "The reviewer should" → "enables"
- "Reviewer B needs to" → "provides"
- "This requires" → "this includes"

**Maintained appropriate structure:**
- Type definitions in A.3: kept structured presentation (creation, purpose, content, roles)
- Tables: kept formatting
- SP file inventories in A.4: kept list format for clarity

**Improved natural flow in A.2:**
- Converted choppy instructional descriptions to smooth narrative
- Integrated graph description elements into flowing prose
- Maintained technical precision while improving readability

**Rationale:**

Appendix serves dual function:
1. Must be navigable as reference material (justifies some structure)
2. Must maintain philosophical paper's careful, precise tone (requires limiting formatting)

Balance achieved through: flowing prose in narrative sections (A.1, A.2, A.5), structured presentation in reference sections (A.3, A.4), minimal formatting throughout.

**Impact:**

Revised Appendix matched paper's style while preserving reference functionality:
- Natural prose in A.2 graph description (no longer reads like drawing instructions)
- Appropriate structure in A.3 type definitions (scannable but not over-formatted)
- Consistent with main paper's scholarly exposition
- Maintained usability for navigation and lookup

## MOD-004: Figure Integration Transformation

**Date:** October 19, 2025

**Issue Identified:**

User requested fundamental restructuring: "insert all writing referring to graphs: (1) in square brackets, clearly distinguishing it from text that will remain (2) as a prompt for Claude to create the image in vectorial form"

**Problem Context:**

A.2's entire "Conceptual Graph" subsection (original ~1,500 words) was written as detailed spatial description intended for someone to sketch from text. Format:
- "Show at foundation: two source nodes..."
- "Draw thick constant arrow from Phase 1..."
- "Use stacked effect to show accumulation..."

This needed transformation into:
1. **Referenced figure in main text** (NOT described spatially)
2. **Separate comprehensive prompt** for vectorial figure creation
3. **Analytical text** discussing what the figure demonstrates (conceptual significance)

**Decision/Resolution:**

**A.2 Transformation:**

Replaced detailed graph description with figure reference:
```
[Figure 1: Document Creation Flow - See Figure 1 for visual representation 
of the complete document creation process across six phases]
```

Rewrote section to **REFERENCE** visual elements rather than **DESCRIBE** them spatially:

**Original (instructional):**
"Show at foundation two source nodes: Epistemic Traces (Type 2) and Complete Prompt development. Draw arrow from Epistemic Traces to Prompt Development Log. Use thick arrow to show this feeds Complete Prompt creation."

**Revised (referential):**
"As shown in Phase 1, Epistemic Traces serve as foundational source material that feeds through Prompt Development Log into Complete Prompt creation. The thick arrow from Phase 1 through Phase 2 shows Complete Prompt as constant input..."

Changed perspective throughout A.2:
- "Show..." → "The diagram shows..."
- "Draw curved feedback arrows..." → "The curved feedback arrows between sections illustrate..."
- "Use stacked visual effect..." → "The stacked visual effects in Phase 2 illustrate..."

**Preserved conceptual explanation and interpretation:**
- Why graph is complex (reflects actual process)
- What each phase demonstrates
- Ecological validity connections
- Navigation guidance

**Removed all spatial/directional drawing instructions** from body text.

**Separate Figure Prompt Creation:**

Created comprehensive Figure 1 prompt (~1,200 words) structured as:

**Overview:** Horizontal flow diagram, six phases, clean academic style
**Design Principles:** Timeline left-to-right, clear phase boundaries, accumulation emphasis
**Phase 1 specifications:** Elements, arrows, annotations, visual notes
**Phase 2 specifications:** Repeating pattern, accumulation visualization, forward movement
**Phase 3 specifications:** Methodology design branch (parallel track)
**Phase 4 specifications:** Later writing with Section 7→8 complexity
**Phase 5 specifications:** Editorial revision with consolidation event
**Phase 6 specifications:** Appendix convergence point
**Legend/Key:** Document types, arrow types, annotations
**Style Guidelines:** Academic appearance, readability, professional palette

All spatial/directional instructions moved here, organized for designer or AI image generation.

**Rationale:**

Academic papers reference figures rather than embedding descriptions in prose. This separation:
- Keeps main text focused on conceptual significance
- Preserves detailed technical specifications for figure creation
- Enables independent work on visualization
- Maintains professional academic presentation style
- Allows A.2 to discuss what the figure reveals without describing how to draw it

**Impact:**

**A.2 transformed** from drawing instructions to proper academic prose:
- References Figure 1 naturally in flowing discussion
- Interprets what the visualization shows
- Discusses conceptual significance of complexity
- Maintains ecological validity framing
- Professional scholarly exposition

**Figure prompt created** as standalone artifact:
- Comprehensive enough for designer to execute
- Structured for AI image generation tools
- All technical specifications preserved
- Usable independently of A.2 text

Division of labor established: A.2 interprets significance, Figure prompt specifies execution.
